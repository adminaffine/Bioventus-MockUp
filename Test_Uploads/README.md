# Test_Uploads

Sample CSV datasets for the **Upload & Analyze** feature, **Data Quality Profiler**, **Integration & Lineage**, and **Data Governance** (integrity and schema drift).

## How to regenerate

From the project root:

```bash
cd backend/data
python seed_test_uploads.py
```

Files are written to `Test_Uploads/` at the project root. Requires `faker`; optional `pii_generators` for PII columns.

## Files and purposes

| File | Rows | Purpose |
|------|------|--------|
| **CUSTOMER_MASTER_One.csv** | 400 | DQ: duplicates, missing email/phone/dob, invalid phone, inconsistent state/segment. Join key for orders and patient support. |
| **CUSTOMER_MASTER_v2.csv** | 80 | **Data Governance / Schema drift:** Same entity as baseline but with `dob` → `date_of_birth`, `customer_segment` removed, `loyalty_tier` added, `zip` as integer. Use with Integrity Monitor schema-drift check. |
| **PRODUCT_CATALOG_One.csv** | 100 | DQ: missing FDA/device_class/HCPCS, past expiry, product name casing, manufacturer whitespace. |
| **SALES_ORDERS_One.csv** | 500 | DQ + Integration: orphan customer_id, ship_date before order_date, negative total_amount, null sales_rep_id. Referential and temporal integrity scenarios. |
| **PATIENT_SUPPORT_One.csv** | 150 | DQ + Compliance: orphan customer/product, MDR/HIPAA gaps, resolution_date before case_date. |

## Data Governance alignment

- **Schema drift:** Upload `CUSTOMER_MASTER_v2.csv` as `customer_master` and call the Integrity API schema-drift endpoint with its column list to see renames, removed/added columns, and pipeline risk.
- **Referential / temporal / domain:** The main seed DB (`backend/data/`) and these test CSVs share similar issue types (orphans, bad dates, domain values); the Integrity Engine and Governance Command Center report on the **seeded database**. Use these CSVs to test **Upload & Analyze** quality and integration checks; use the app’s **Data Governance** page for full integrity and trust scores against the DB.

## Manifest

`dataset_manifest.json` is generated by `seed_test_uploads.py` and lists each file, columns, embedded issue counts, and join keys for integration checks.
